    Create `src/llm/health.py`. Do NOT modify client.py or config.py.

    Define:

    ```python
    @dataclass
    class HealthResult:
        provider: str
        model: str
        status: str            # "OK" | "UNREACHABLE"
        error: str | None
        embeddings_status: str # "OK" | "UNREACHABLE" | "not configured"
        embeddings_error: str | None
    ```

    Define `async def check_health(config: Config) -> HealthResult`:

    1. Probe LLM provider:
       - Construct client = make_llm_client(config)
       - Call: await asyncio.wait_for(client.chat([{"role": "user", "content": "ping"}]), timeout=5.0)
       - On success: status="OK", error=None
       - On LLMError or asyncio.TimeoutError: status="UNREACHABLE", error=str(e)
       - On any other exception: status="UNREACHABLE", error=repr(e)

    2. Probe embeddings (if config.embeddings is not None):
       - Call: await asyncio.wait_for(client.embed(["ping"]), timeout=5.0)
       - On success (non-empty list returned): embeddings_status="OK", embeddings_error=None
       - On LLMError or asyncio.TimeoutError: embeddings_status="UNREACHABLE", embeddings_error=str(e)
    3. If config.embeddings is None: embeddings_status="not configured", embeddings_error=None

    4. Return HealthResult(
         provider=config.llm.provider,
         model=config.llm.model,
         status=status,
         error=error,
         embeddings_status=embeddings_status,
         embeddings_error=embeddings_error,
       )

    check_health() MUST NOT raise under any circumstances — all errors are captured
    and returned in the HealthResult fields.

    Use structlog.get_logger(__name__) for debug logging. Do NOT use standard logging.
