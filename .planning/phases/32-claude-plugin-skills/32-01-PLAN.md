Create `src/cli/commands/install_cmd.py` with two internal helpers and the typer command.

**`install_mcp_global(force: bool = False) -> dict`:**
- Reads `~/.claude/settings.json` (creates empty dict if missing or malformed)
- Merges `mcpServers.recall = {"command": "recall", "args": ["mcp", "serve"]}` into the config
- If `"recall"` already in `config["mcpServers"]` and not force → skip, return `{"mcp_registered": False}`
- Write back to `~/.claude/settings.json` (mkdir parents=True)
- Return `{"mcp_registered": True}`
- Use `structlog.get_logger(__name__)` for logging (NOT stdlib logging — this is not in mcp_server/)

**`ensure_recall_dir() -> bool`:**
- `recall_dir = Path.home() / ".recall"`
- If `recall_dir` does not exist → create it, return True
- If exists → do NOT touch `config.toml` regardless — return False
- No files written by this function — directory creation only (per INST-03: existing config.toml left unchanged)

**`install_command(force: bool = False)`** (typer function, registered as `install`):
```python
def install_command(
    force: Annotated[bool, typer.Option("--force", help="Overwrite existing MCP entry")] = False,
) -> None:
    """Install recall as a Claude plugin: register MCP server and create ~/.recall/ directory."""
```
Steps:
1. Call `ensure_recall_dir()` — print success if created, dim skip if already existed
2. Call `install_mcp_global(force)` — print success if registered, dim skip if already present
3. Print final summary: "recall plugin installed. Start Claude and type /recall-setup to configure."

Do NOT write config.toml in this command — that is the job of the `/recall-setup` skill.
Do NOT call `install_global_hooks()` here — that is `recall init`'s job.

Wire into CLI: in `src/cli/__init__.py`, add:
