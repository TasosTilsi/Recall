```python
app = typer.Typer(name="search", help="Search the knowledge graph")

@app.callback(invoke_without_command=True)
def search(
    ctx: typer.Context,
    query: str = typer.Argument(..., help="Search query"),
    semantic: bool = typer.Option(False, "--semantic", help="Use vector/embeddings search"),
    related: bool = typer.Option(False, "--related", help="Include one hop of backlinked entities"),
):
    ...
```

Implementation rules:

1. **Import** `load_config` from `src/llm/config.py` and `GraphService` from `src/graph/service.py`.

2. **Embeddings check** — when `--semantic` is passed, call `load_config()` and verify it has an embeddings section. If embeddings are not configured, print exactly:
   ```
   Semantic search requires [embeddings] in ~/.recall/config.toml
   ```
   then call `raise typer.Exit(code=1)`.

3. **FTS search** (default, no `--semantic`): call `service.fts_search(query)`. Each result must be displayed as:
   ```
   [<entity_type>] <name>
     <snippet>
     sha: <commit_sha>  date: <date>
   ```

4. **Semantic search** (`--semantic`): call `service.semantic_search(query)`. Same display format as FTS.

5. **Related** (`--related`): after each primary result, call `service.get_related(entity_id)` and print each related entity indented under its parent, prefixed with the relationship type:
   ```
   [<entity_type>] <name>
     <snippet>
     sha: <commit_sha>  date: <date>
     -> <relation_type>: [<related_type>] <related_name>
   ```

6. **No results**: if the results list is empty, print `No results found for: <query>` and exit 0.

7. **Error handling**: wrap the entire body in try/except, log errors via structlog, exit 1 on exception.

```python
import typer
import toml
from pathlib import Path
import structlog

app = typer.Typer(name="config", help="Inspect and modify recall configuration")

CONFIG_PATH = Path.home() / ".recall" / "config.toml"
```

Implement three subcommands:

1. **`show` subcommand**:
   ```python
   @app.command()
   def show():
       """Print full config file contents."""
   ```
   - Read `CONFIG_PATH` with `CONFIG_PATH.read_text()` and `typer.echo()` the result.
   - If file does not exist, print `Config file not found: {CONFIG_PATH}` and exit 1.

2. **`get` subcommand**:
   ```python
   @app.command()
   def get(key: str = typer.Argument(..., help="Dotted key, e.g. llm.provider")):
       """Get a config value by dotted key."""
   ```
   - Load config with `toml.loads(CONFIG_PATH.read_text())`.
   - Traverse the dict using `key.split(".")`.
   - If any segment is missing, print `Key not found: <key>` and `raise typer.Exit(code=1)`.
   - Otherwise `typer.echo(value)`.

   ```python
   @app.command(name="set")

   ):
       """Set a config value by dotted key."""
   ```

   - Traverse/create nested dicts using `key.split(".")`.
   - Write back with `toml.dumps(config)` to `CONFIG_PATH`.
   - Print `Set <key> = <value>`.

All output via `typer.echo()`. Wrap each command body in try/except with structlog error logging and `raise typer.Exit(code=1)` on failure.
