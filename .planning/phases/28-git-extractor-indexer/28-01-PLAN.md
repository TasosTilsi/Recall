<task type="auto" tdd="true">
  <name>Task 1: Git walker — walk_commits, batch_commits, fetch_diff</name>
  <files>src/extractor/__init__.py, src/extractor/git_walker.py, tests/extractor/test_git_walker.py</files>
  <behavior>
    - CommitRecord is a dataclass with fields: sha (str), short_sha (str), author (str), date (datetime), message (str), diff (str)
    - walk_commits(repo_root: Path) -> list[CommitRecord] yields commits oldest-first (reverse=True in GitPython); skips merge commits where message starts with "Merge "
    - batch_commits(commits: list[CommitRecord], batch_size: int = 10) -> list[list[CommitRecord]] splits list into batches; last batch may be smaller
    - fetch_diff(commit, repo) -> str returns diff text truncated to 4000 chars; returns empty string on GitCommandError
    - batch_commits([], 10) returns []
    - batch_commits([c1, c2, c3], 2) returns [[c1, c2], [c3]]
  </behavior>
  <action>
    Create `src/extractor/__init__.py` as empty package marker.

    Create `src/extractor/git_walker.py`:
    - Import: `git`, `structlog`, `dataclasses`, `datetime`, `pathlib.Path`
    - `CommitRecord` dataclass: sha, short_sha (sha[:7]), author (commit.author.name), date (commit.authored_datetime), message (commit.message.strip()), diff (str)
    - `fetch_diff(commit, repo) -> str`: calls `repo.git.diff(commit.parents[0].hexsha, commit.hexsha)` if commit.parents else `repo.git.show(commit.hexsha, "--stat")`. Truncate to 4000 chars. Wrap in try/except git.GitCommandError, return "" on error.
    - `walk_commits(repo_root: Path) -> list[CommitRecord]`: opens `git.Repo(repo_root, search_parent_directories=True)`, iterates `repo.iter_commits(rev="HEAD", reverse=True)`, skips commits where message starts with "Merge ", builds CommitRecord for each. Logs count at end via structlog.
    - `batch_commits(commits: list[CommitRecord], batch_size: int = 10) -> list[list[CommitRecord]]`: standard list chunking.

    Create `tests/extractor/__init__.py` and `tests/extractor/test_git_walker.py` with pytest tests using the current repo (Path.cwd()) as test data — assert walk_commits returns a non-empty list, all records have non-empty sha and message, batch_commits edge cases.
  </action>
  <verify>
    <automated>pytest tests/extractor/test_git_walker.py -x -q</automated>
  </verify>
  <done>walk_commits returns CommitRecord list from real repo, batch_commits splits correctly, all tests pass</done>
</task>

<task type="auto" tdd="true">
  <name>Task 2: Prompt builder — build_batch_prompt, ExtractionResult schema</name>
  <files>src/extractor/prompt.py, tests/extractor/test_prompt.py</files>
  <behavior>
    - EXTRACTION_SCHEMA is a dict (JSON Schema) describing the expected LLM response structure: {"entities": [{"type": str, "name": str, "content": str, "commit_sha": str}]}
    - Valid entity types are exactly: decision, bug_fix, pattern, file, concept, tech_debt
    - EntityRecord is a TypedDict with: type (str), name (str), content (str), commit_sha (str)
    - ExtractionResult is a TypedDict with: entities (list[EntityRecord])
    - build_batch_prompt(batch: list[CommitRecord]) -> str returns a prompt string containing each commit's sha[:7], author, message, and diff (truncated at 800 chars per commit to manage total prompt size)
    - The prompt instructs the LLM to return ONLY valid JSON matching EXTRACTION_SCHEMA — no prose, no markdown code fences
    - build_batch_prompt([]) raises ValueError("batch must not be empty")
    - The returned prompt string contains the word "JSON" and lists the six entity types
  </behavior>
  <action>
    Create `src/extractor/prompt.py`:

    Define `VALID_ENTITY_TYPES = {"decision", "bug_fix", "pattern", "file", "concept", "tech_debt"}`.

    Define `EXTRACTION_SCHEMA` as a plain dict describing the response shape (embed as a Python constant, also render as JSON in the prompt so the LLM sees it).

    Define `EntityRecord` and `ExtractionResult` as TypedDicts.

    Implement `build_batch_prompt(batch: list[CommitRecord]) -> str`:
    - Raises ValueError if batch is empty
    - Formats each commit as:
      ```
      --- Commit {sha[:7]} by {author} ---
      Message: {message}
      Diff (truncated):
      {diff[:800]}
      ```
    - Wraps in a system prompt that says: "You are a code knowledge extractor. Analyze the following git commits and return ONLY valid JSON with this exact structure: {EXTRACTION_SCHEMA as JSON}. Entity types must be one of: decision, bug_fix, pattern, file, concept, tech_debt. Entity names must be lowercase and trimmed. Do not include any prose, markdown, or code fences — return raw JSON only."
    - Returns the complete prompt string.

    Create `tests/extractor/test_prompt.py`: test empty batch raises ValueError, non-empty batch contains commit shas and "JSON", schema dict has correct keys.
  </action>
  <verify>
    <automated>pytest tests/extractor/test_prompt.py -x -q</automated>
  </verify>
  <done>build_batch_prompt produces well-formed prompt containing all six entity types and commit data, tests pass</done>
