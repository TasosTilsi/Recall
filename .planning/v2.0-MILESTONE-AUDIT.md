**Found:** `scripts/verify_phase_10.py`, `scripts/verify_phase_11.py`, `scripts/verify_phase_12.py`, `scripts/verify_phase_13.py` all reference `graphiti` binary (e.g., `GRAPHITI = str(ROOT / ".venv" / "bin" / "graphiti")`) — tool was renamed to `recall` in Phase 16.
**Impact:** Verify scripts fail if run against the current codebase (binary not found).
**Assigned to:** Phase 24-01
**Status:** Addressed in Phase 24-01

### Gap 4: EntityPanel Always Showed "Normal" Status

**Found:** `/api/detail` route only checked `pinned: bool` — did not compute full `retention_status`. `EntityPanel.retentionStatus()` only checked `entity.pinned` — did not read `retention_status` field. Result: entity detail panel always showed "Normal" or "Pinned" status, never "Stale" or "Archived".
**Impact:** UI-03 (retention filter) partially broken — filter dropdown and graph rings worked, but entity detail panel status badge was wrong.
**Resolved by:** Phase 22-01 (`fb70e4a`) — added retention_status computation to `/api/detail` entity branches; Phase 22-02 (`b6aa10c`) — wired `EntityPanel.retentionStatus()` to read `entity.retention_status` directly.
**Status:** RESOLVED — TypeScript build clean; 21 pytest tests pass; EntityPanel reads field correctly

---

### Gap 5: Graph.tsx Node-Click Did Not Open DetailPanel

**Found:** `ui/src/routes/Graph.tsx` had a `selectedNode` state and `handleNodeClick` callback, but the `selectedNode` was suppressed with `void selectedNode;` rather than wired to render `<DetailPanel>`. Clicking a node in the graph view had no effect on the UI.
**Impact:** Graph view was read-only — users could not click a node to see its detail panel.
**Resolved by:** Phase 24-02 (`85cef3e`) — removed `void selectedNode`; changed state type to `PanelItem`; updated `handleNodeClick` to create `PanelItem`; added `<DetailPanel>` to JSX return.
**Status:** RESOLVED — Graph.tsx imports and renders DetailPanel; TypeScript types verified
