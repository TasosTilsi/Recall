### Gap 2 — `_GRAPHITI_CLI` Stale Binary in worker.py

**File:** `src/queue/worker.py` line 30
**Code:** `_GRAPHITI_CLI = str(Path(sys.executable).parent / "graphiti")`
**Impact:** The `graphiti` binary was removed in Phase 16. Any `cli_replay` job type that falls through to `_execute_cli_command` (line 303) will fail with FileNotFoundError at runtime.

**Mitigating factor:** The primary queue path (`capture_tool_use` jobs) uses `_handle_capture_tool_use()` → `get_service().add()` directly and does NOT call `_GRAPHITI_CLI`. The broken path only fires for `cli_replay` job types. Phase 16 Plan 03 updated `manager.py` to `_RECALL_CLI` but missed `worker.py`.

**Fix:** One-line change — `_GRAPHITI_CLI = str(Path(sys.executable).parent / "recall")`

**Affects:** MEM-02 (partial)

---

### Gap 3 — MCP Server Broken (`src/mcp_server/tools.py`)

**File:** `src/mcp_server/tools.py` lines 26, 63, 427; `src/mcp_server/context.py` line 17
**Code:** `_GRAPHITI_CLI = str(Path(sys.executable).parent / "graphiti")`
**Impact:** All MCP tool handlers that invoke CLI subcommands (`graphiti_search`, `graphiti_index`, `graphiti_capture`) will fail. `graphiti_capture` additionally calls a deleted `capture` command.

**Note:** MCP server is outside the 5-phase milestone scope but is a live runtime break that will affect any user who runs `recall mcp serve`.

**Fix:** Update `_GRAPHITI_CLI` to `_RECALL_CLI` pointing to `recall` binary; update `graphiti_capture` to call the `note` command instead of deleted `capture`.
