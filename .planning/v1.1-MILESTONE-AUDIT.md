### Integration Gaps Found (by gsd-integration-checker)

#### INT-01 — Hard Bug: `RetentionManager.get_node_metadata` Does Not Exist
`routes.py:118` calls `retention.get_node_metadata(uuid=uuid, scope=scope_key)`. Method never existed. `AttributeError` caught by `try/except Exception: meta = None`. Every `/api/nodes/{uuid}` response returns `pinned: False, accessCount: 0, lastAccessedAt: ""` — regardless of actual state.

**Why tests don't catch it:** `test_node_detail_endpoint` only asserts `id`, `name`, `entityType` — never the retention fields.

**Fix:**
```python
access = retention.get_access_record(uuid=uuid, scope=scope_key)
meta = {"last_accessed_at": access["last_accessed_at"], "access_count": access["access_count"],
        "pinned": retention.is_pinned(uuid=uuid, scope=scope_key)}
```

#### INT-02 — Design Gap: Archived Nodes Visible in UI Graph Canvas
`list_entities_readonly()` queries Kuzu without consulting `retention.db`. Nodes archived by `compact --expire` remain visible in the UI canvas. `list_entities()` applies `get_archive_state_uuids()` post-filter — `list_entities_readonly()` does not.

**Fix:** Add `get_archive_state_uuids()` filter to `list_entities_readonly()`.

#### INT-03 — Design Gap: Pin State Hardcoded False in Graph Canvas
All node dicts from `list_entities_readonly()` contain `"pinned": False`. Pinned nodes get no visual distinction in the canvas.

**Fix:** Add `get_pin_state_uuids()` lookup alongside the archive filter.

#### INT-04 — Missing: `[ui]` Section Not in `graphiti config show/set`
`LLMConfig.ui_port` and `ui_api_port` not in `VALID_CONFIG_KEYS`. Users must edit `~/.graphiti/llm.toml` manually to change the UI port.

**Fix:** Add `ui.port` and `ui.api_port` to `VALID_CONFIG_KEYS`, `attr_map`, table, and JSON output in `config.py`.
