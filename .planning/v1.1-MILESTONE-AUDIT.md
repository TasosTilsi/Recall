### Critical Integration Bug

**`routes.py:118` — `retention.get_node_metadata()` does not exist**

The `GET /api/nodes/{uuid}` endpoint attempts to enrich node detail responses with retention metadata by calling `retention.get_node_metadata(uuid=uuid, scope=scope_key)`. This method is not implemented on `RetentionManager`.

The `AttributeError` is silently caught by a `try/except Exception: meta = None` block, causing the endpoint to return `200 OK` with stale defaults (`last_accessed_at: ""`, `access_count: 0`, `pinned: false`) instead of live retention sidecar data.

**Fix:** Replace with two existing API calls:
```python
access = retention.get_access_record(uuid=uuid, scope=scope_key)
pinned = retention.is_pinned(uuid=uuid, scope=scope_key)
meta = {"last_accessed_at": access["last_accessed_at"], "access_count": access["access_count"], "pinned": pinned}
```

**Impact:** Retention data not shown in UI node detail view. Core requirements (CLI pin/unpin/stale/compact) unaffected — all fully operational.
