QUESTION 2: GraphProvider.LADYBUG exists?
=========================================================

Command run:
  python3 -c "from graphiti_core.driver.kuzu_driver import GraphProvider; print(list(GraphProvider))"

Output:
  [<GraphProvider.NEO4J: 'neo4j'>, <GraphProvider.FALKORDB: 'falkordb'>,
   <GraphProvider.KUZU: 'kuzu'>, <GraphProvider.NEPTUNE: 'neptune'>]

ANSWER: NO — GraphProvider.LADYBUG does NOT exist in graphiti-core==0.28.1.
The enum only contains: NEO4J, FALKORDB, KUZU, NEPTUNE.
Plan 02 must either:
  (a) Add LADYBUG = "ladybug" to the local vendor copy, OR
  (b) Use GraphProvider.KUZU as an alias (since LadybugDB is a Kuzu fork, Cypher dialect is identical), OR
  (c) Call FTS initialization directly without relying on GraphProvider.LADYBUG
The recommended approach is (b) for minimal vendoring — pass GraphProvider.KUZU to any
method that needs a provider, since the Cypher/FTS schema is identical.

=========================================================
QUESTION 3: lb.Database() works?
=========================================================

Command run:
  python3 -c "
  import real_ladybug as lb
  import tempfile, os
  with tempfile.TemporaryDirectory() as d:
      db_path = os.path.join(d, 'test.lbdb')
      db = lb.Database(db_path)
      conn = lb.Connection(db)
      result = conn.execute('RETURN 1 AS x')
      print('SUCCESS:', list(result.rows_as_dict()))
      conn.close()
  "

Output:
  SUCCESS: [{'x': 1}]

ANSWER: YES — lb.Database(path) creates a fresh .lbdb file successfully.
Connection API: lb.Connection(db).execute(cypher) returns a QueryResult
with .rows_as_dict() method — identical interface to kuzu.Connection.
This means test_storage.py's raw connection tests can swap kuzu.Connection
for lb.Connection with zero logic changes.

QUESTION 4: AsyncConnection available?
=========================================================

Command run:
  python3 -c "import real_ladybug as lb; print(hasattr(lb, 'AsyncConnection'))"

Output:
  True

Command run:
  python3 -c "import real_ladybug as lb; print([x for x in dir(lb) if 'Async' in x or 'async' in x])"

Output:
  ['AsyncConnection', 'async_connection']

AsyncConnection signature:
  lb.AsyncConnection(database, max_concurrent_queries=4, max_threads_per_query=0)

Key methods on AsyncConnection:
  - acquire_connection() -> Connection  (synchronous, from pool)
  - release_connection(conn)
  - async execute(query, parameters=None) -> QueryResult | list[QueryResult]
  - close()
  - context manager support (__enter__/__exit__)

ANSWER: YES — AsyncConnection is available and provides async execute() method.
This is the primary connection type to use in LadybugDriver (like KuzuDriver uses
kuzu.AsyncConnection).

=========================================================
CONCLUSION: Vendoring scope
=========================================================

Based on spike results:

1. LadybugDriver MUST be vendored locally (~280 lines):
   - real_ladybug provides only low-level bindings (Database, Connection, AsyncConnection)
   - No graphiti-compatible driver exists in the package
   - Must implement: LadybugDriver class with execute_query(), setup_schema(),
     retrieve_episodes(), build_indices_and_constraints(), with_database(), clone()
   - The driver wraps lb.Database (stored as self.db) + lb.AsyncConnection for queries

2. GraphProvider.LADYBUG does NOT exist — use GraphProvider.KUZU as alias:
   - LadybugDB is a Kuzu fork with identical Cypher dialect and FTS schema
   - Passing GraphProvider.KUZU to get_fulltext_indices() will return the correct
     FTS index definitions for LadybugDB
   - No enum patching required

3. .lbdb file creation works cleanly:
   - lb.Database(path) creates directory-based DB at given path
   - Connection API is identical to kuzu: rows_as_dict(), execute() return same types
   - Safe to rename DB suffix from .kuzu to .lbdb (different format; do NOT try to open old files)

4. AsyncConnection is available and needed for LadybugDriver:
   - Use lb.AsyncConnection(db) as the async execution engine in LadybugDriver
   - Wrap with acquire_connection()/release_connection() for sync compatibility

VENDORING SCOPE: MINIMAL (~280 lines)
  File: src/storage/ladybug_driver.py
  Key: Vendor only LadybugDriver class (no operation handler classes needed)
  The real_ladybug bindings handle all low-level DB operations; we just need
  the graphiti-compatible wrapper that satisfies graphiti_core's driver protocol.
